# <a name="c6215"></a>C6215
警告 C6215：语义不同的整数类型之间的强制转换： Boolean 类型转换为 HRESULT

 此警告表明布尔值被强制转换为 @no__t 0。 布尔类型通过非零值指示成功，而 `HRESULT` 中的成功（@no__t）表示值0。 将 Boolean 类型转换为 @no__t 0，然后在测试表达式中使用它会产生不正确的结果。

 当使用布尔值作为 @no__t 0 或 `FAILED` 宏的参数时，通常会出现此警告，这会将其自变量显式转换为 `HRESULT`。

## <a name="example"></a>示例
 下面的代码将生成此警告：

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

void f( REFGUID riid1, REFGUID riid2 )
{
  if (SUCCEEDED( IsEqual( riid1, riid2 ) )) //warning 6215
  {
    // success code ...
  }
  else
  {
    // failure code ...
  }
}
```

 通常，@no__t 0 或 `FAILED` 宏只能应用于 `HRESULT`。

 若要更正此警告，请使用以下代码：

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

void f( REFGUID riid1, REFGUID riid2 )
{
  if (IsEqual( riid1, riid2 ) == TRUE)
  {
    // code for riid1 == riid2
  }
  else
  {
    // code for riid1 != riid2
  }
}
```

 有关详细信息，请参阅[SUCCEEDED 宏](http://go.microsoft.com/fwlink/?LinkId=92738)和[FAILED 宏](http://go.microsoft.com/fwlink/?LinkId=180875)
