# <a name="c26438-nogoto"></a>C26438 NO_GOTO

"避免 goto。"

**C++核心准则**:ES.76:避免 goto

Goto 的使用被众所周知的危险且容易出错的做法是。 仅在生成的代码 （例如在生成的语法的分析器） 中可接受它。 提供新式C++功能和实用工具提供的准则支持库很轻松地完全避免 goto。

## <a name="remarks"></a>备注

- 此规则警告上的 goto 的所有事件，即使它发生在死代码，这是永远不会使用，因此由编译器忽略模板代码除外。
  - 警告可能会干扰性，如果它们在遇到包含 goto 的宏。 当前的报告机制将指向此类宏获取展开其中的所有实例。 但解决方法通常在一个位置通过更改宏或避免使用它，并利用更多的可维护性机制完成。

## <a name="example"></a>示例

goto cleanup' 中宏

```cpp
#define ENSURE(E, L) if (!(E)) goto L;

void poll(connection &c)
{
    ENSURE(c.open(), end);                  // C26438

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        ENSURE(c.read_header(h), end);      // C26438
        ENSURE(c.read_signature(s), end);   // C26438
        // ...
    }

end:
    c.close();
}
```

goto cleanup' 宏中-替换为 gsl:: finally

```cpp
void poll(connection &c)
{
    auto end = gsl::finally([&c] { c.close(); });

    if (!c.open())
        return;

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        if(!c.read_header(h))
            return;
       if(!c.read_signature(s))
            return;
        // ...
    }
}
```
