# <a name="c6067"></a>C6067
警告 C6067： > 参数 \<number 调用 \<function > 必须是字符串的地址

 此警告表明格式说明符和函数参数不匹配。 即使警告建议使用字符串地址，也必须检查函数要求的参数类型，然后才能更正问题。 例如，`printf` 的 `%s` 规范需要一个字符串参数，但 `scanf` 中的 `%s` 规范需要字符串的地址。

 此缺陷可能会导致某种形式的崩溃或损坏。

## <a name="example"></a>示例
 下面的代码将生成此警告，因为传递的是整数而不是字符串：

```
#include <stdio.h>
void f_defective( )
{
  char *str = "Hello, World!";
  printf("String:\n %s",1); // warning
  // code ...
}
```

 若要更正此警告，请将字符串作为参数传递到 `printf`，如以下代码所示：

```
#include <stdio.h>
void f_corrected( )
{
  char *str = "Hello, World!";
  printf("String:\n %s",str);
  // code ...
}
```

 下面的代码将生成此警告，因为在将参数、缓冲区传递到 `scanf` 时指定了错误的间接级别：

```
#include <stdio.h>
void h_defective( )
{
  int retval;
  char* buffer = new char(20);
  if (  buffer )
  {
    retval = scanf("%s", &buffer); // warning C6067
    // code...
    delete buffer ;
  }
}
```

 若要更正以上警告，请传递正确的参数，如以下代码所示：

```
#include <stdio.h>
void h_corrected( )
{
  int retval;
  char* buffer = new char(20);
  if ( buffer )
  {
    retval = scanf("%s", buffer);
    // code...
    delete buffer;
  }
}
```

 下面的代码使用安全的字符串操作函数来更正此警告：

```
#include <stdio.h>
void f_safe( )
{
  char buff[20];
  int retVal;

  sprintf_s( buff, 20, "%s %s", "Hello", "World!" );
  printf_s( "String:\n   %s  %s", "Hello", "World!" );
  retVal = scanf_s("%s", buff, 20);
}
```
