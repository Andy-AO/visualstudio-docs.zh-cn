# <a name="c26409-no_new_delete"></a>C26409 NO_NEW_DELETE
即使代码清除对 malloc （）和 free （）的调用，我们仍建议您考虑更好的选项，而不是显式使用运算符[new 和 delete](/cpp/cpp/new-and-delete-operators)。

  核心指导原则： **C++** @NO__T 0R：避免显式调用 new 和 delete @ no__t-0

最终的解决方法是开始使用具有相应工厂函数（如[std：： make_unique](/cpp/standard-library/memory-functions#make_unique)）的智能指针。

## <a name="remarks"></a>备注
- 该检查器发出警告，对任何类型的运算符 `new` 或 `delete`：标量、矢量、重载版本（全局和类特定）以及放置版本发出警告。 后一种情况下，可能需要对核心指导原则进行一些说明，如建议的修补程序，以后可能会忽略这些问题。
