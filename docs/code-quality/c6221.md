# <a name="c6221"></a>C6221
警告 C6221:语义不同的整数类型之间的隐式强制转换： 将 HRESULT 与整数进行比较。 请考虑改用 SUCCEEDED 或 FAILED 宏

 此警告意味着`HRESULT`要进行比较的非零的整数。 在成功完成`HRESULT`(`S_OK`) 由 0 表示。 因此，进行隐式转换的`HRESULT`为整数，则将生成不正确的值，并且可能会导致错误的结果。 它通常由导致错误地期望将函数时要返回一个整数，它会实际返回`HRESULT`。

## <a name="example"></a>示例
 下面的代码通过比较生成此警告`HRESULT`与整数值：

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == 4)
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 若要更正此警告，下面的代码使用`FAILED`宏：

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 此警告，请`SCODE`类型等效于`HRESULT`。

 有关详细信息，请参阅[SUCCEEDED 宏](http://go.microsoft.com/fwlink/?LinkId=92738)并[FAILED 宏](http://go.microsoft.com/fwlink/?LinkId=180875)。

 请注意，使用 malloc 和可用 （和相关的动态内存分配 Api） 有内存泄漏和异常方面的许多缺陷。 若要避免这些类型的泄漏和异常问题完全，使用的机制，通过提供C++标准模板库 (STL)。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)， [unique_ptr](/cpp/standard-library/unique-ptr-class)，并[向量](/cpp/standard-library/vector)。 有关详细信息，请参阅[智能指针](/cpp/cpp/smart-pointers-modern-cpp)并[C++标准库](/cpp/standard-library/cpp-standard-library-reference)。
