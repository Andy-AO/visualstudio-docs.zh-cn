# <a name="c6284"></a>C6284
警告 C6284：作为参数 '% d ' 传递的对象在调用 \<function > 时需要字符串。

 此警告表明，格式字符串指定一个字符串，例如，`printf` 的 @no__t 规范或 `scanf`，但已传递C++对象。

 此缺陷可能会产生不正确的输出或故障。

 此消息通常是由于将实现某些字符串C++类型的对象（例如 `std::string`、`CComBSTR` 或 `bstr_t`）传递到 C `printf` 的调用而导致的。 根据C++类的实现，即，如果定义了正确的强制转换运算符，则只要需要C++ C 字符串，就通常可以透明地使用字符串对象;但是，因为 `printf` 样式函数的参数实质上是非类型化的，所以不会转换为字符串。

 根据对象的不同，将 `static_cast` 运算符插入到适当的字符串类型（例如，`char *` 或 `TCHAR``*`）或调用成员函数（例如，`c_str()`，在 @no__t 的实例上）可能会非常合适。

## <a name="example"></a>示例
 下面的代码将生成此警告，因为 `CComBSTR` 会传递到 `sprintf` 函数：

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf(buff,"%ws",bstrValue);
}
```

 下面的代码使用静态强制转换来更正此警告：

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));
}
```
