# <a name="c6101"></a>C6101
警告 C6101：返回未初始化的内存

 通过此函数的成功路径未设置命名的 `_Out_` 参数。 此消息基于 SAL 批注而生成，该批注指明相关函数始终成功。 不返回成功/失败指示的函数应设置其所有 `_Out_` 参数，因为在调用函数之前，分析器假定 @no__t 的参数未初始化数据，并且函数将设置参数，以便它不再处于未初始化状态。 如果函数指示成功/失败，则在失败时不需要设置 `_Out_` 参数，并且可以检测和避免未初始化的位置。 在任何情况下，目标都是避免读取未初始化的位置。 如果该函数有时不触及随后使用的 @no__t 0 参数，则应在调用函数之前初始化参数，并使用 `_Inout_` 批注进行标记，或在适当的情况下使用更明确的 `_Pre_null_` 或 @no__t。 “部分成功”可以使用 `_When_` 批注来处理。 有关详细信息，请参阅[使用 SAL 注释减少 C/C++代码缺陷](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)。
