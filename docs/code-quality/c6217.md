# <a name="c6217"></a>C6217

警告 C6217：语义不同的整数类型之间的隐式强制转换：用 "not" 测试 HRESULT。 请考虑改用[SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded)或[FAILED](/windows/desktop/api/winerror/nf-winerror-failed)宏。

此警告意味着使用 not （@no__t）运算符测试 @no__t 0。 @No__t-1 中的成功（`S_OK`）表示为值0。 但是，0指示布尔类型的失败。 测试 `HRESULT` 替换为 not 运算符（`!`）以确定要运行的代码块可能会导致以下错误的代码路径。 这会导致意外的结果。

## <a name="example"></a>示例

下面的代码将生成此警告，因为 not 运算符用于确定 @no__t 0 值的成功或失败。 在这种情况下，将执行错误的代码路径，因为 @no__t 运行故障代码：

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (!hr)
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

若要更正此警告，以下代码使用 `FAILED` 宏来查找失败：

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (FAILED(hr))
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

对于此警告，@no__t 0 类型等效于 `HRESULT`。

在将 HRESULT （`S_OK`）测试为布尔值时，该变量的典型成功值 @no__t 为-1。

若要验证 @no__t 0 是否成功，请改用 @no__t 的宏。
