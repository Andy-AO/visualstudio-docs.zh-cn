# <a name="c6235"></a>C6235
警告 C6235：（@no__t > &#124; &#124; \<expression >）始终为非零常量

 此警告意味着在测试上下文中发生的逻辑或运算的左侧检测到非零常数值，而不是一个。 不计算逻辑 "或" 运算的右侧，因为生成的表达式的计算结果始终为 true。 这称为 "短路计算"。

 非零常数值，而不是一个，则表示可能已预期按位与运算符（`&`）。 如果非零常量为1，则不会生成此警告，因为它使用有选择地启用代码路径，但如果非零常量的计算结果为1（例如 1 + 0），则会生成此警告。

## <a name="example"></a>示例
 下面的代码将生成此警告，因为 @no__t 为2：

```
#define INPUT_TYPE 2
void f(int n)
{
   if(INPUT_TYPE || n) //warning 6235 issued
   {
      puts("Always gets here");
   }
   else
   {
      puts("Never gets here");
   }
}
```

 下面的代码使用按位与（@no__t 0）运算符来更正此警告：

```
#define INPUT_TYPE 2
void f(int n)
{
   if((INPUT_TYPE & n) == 2)
   {
      puts("bitwise-AND comparison true");
   }
   else
   {
      puts("bitwise-AND comparison false");
   }
}
```
