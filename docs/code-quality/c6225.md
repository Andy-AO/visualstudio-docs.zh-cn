# <a name="c6225"></a>C6225

警告 C6225:语义不同的整数类型之间的隐式强制转换： 将 1 或 TRUE 为 HRESULT。 请考虑改用 S_FALSE

此警告意味着`HRESULT`正在分配或使用显式 1 的值初始化。 布尔值类型的一个非零值; 指示已成功成功 (`S_OK`) 中`HRESULT`由值为 0。 此警告通常由意外混淆的布尔值和`HRESULT`类型。 若要指示成功，符号常量`S_OK`应使用。

## <a name="example"></a>示例

在下面的代码中，分配`HRESULT`将生成此警告：

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    // code ...
    hr = S_OK;
  }
  else
  {
    // code ...
    hr = 1;
  }
}
```

若要更正此警告，请使用以下代码：

```cpp
VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    hr = S_OK;
    // code ...
  }
  else
  {
    hr = S_FALSE;
    // code ...
  }
}
```

此警告，请`SCODE`类型等效于`HRESULT`。

若要指示失败， `E_FAIL`，或应使用另一个常量。

有关详细信息请参阅以下主题之一：

[SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded)

[失败](/windows/desktop/api/winerror/nf-winerror-failed)

若要利用新式C++内存分配方法，使用的机制，提供的C++标准模板库 (STL)。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)， [unique_ptr](/cpp/standard-library/unique-ptr-class)，并[向量](/cpp/standard-library/vector)。 有关详细信息，请参阅[智能指针](/cpp/cpp/smart-pointers-modern-cpp)并[C++标准库](/cpp/standard-library/cpp-standard-library-reference)。
