# <a name="c6387"></a>C6387
警告 C6387： > @no__t @no__t >：这不符合函数的规范 \<function name >：行： x，y

 如果向带批注的函数参数传递意外值，则会引发此警告。 例如，将可能的 null 值传递给使用 `_In_` 批注标记的参数时，将会生成此警告。

## <a name="example"></a>示例
 下面的代码将生成此警告，因为 null 参数传递到 `f(char *)`：

```cpp

#include <sal.h>

_Post_ _Null_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh); // Warning C6387
}
```

 若要更正此警告，请使用以下代码：

```cpp

#include <sal.h>

_Post_ _Notnull_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh);
}
```
