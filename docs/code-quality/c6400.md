# <a name="c6400"></a>C6400
警告 C6400：使用 @no__t 0function name >，可 @no__t 名称 > 中执行不区分大小写的比较。 在非英语区域设置中产生意外结果

 此警告表明，对常量字符串的不区分大小写的比较是以与区域设置相关的方式进行的，在这种情况下，这种比较与区域设置无关。

 此缺陷的典型结果是非英语语言区域设置中的错误行为。 例如，在土耳其语中，".gif" 将不匹配 ""。GIF ";在越南语中，"登录名" 将与 "LOGIN" 不匹配。

 通常应通过 @no__t 的函数执行字符串比较。 若要在 Windows XP 上执行与区域设置无关的比较，第一个参数应为常量 `LOCALE_INVARIANT`。

## <a name="example"></a>示例
 下面的代码将生成此警告：

```
#include <windows.h>
int f(char *ext)
{
  // code...
  return (lstrcmpi(ext, TEXT("gif")) == 0);
}
```

 若要更正此警告，请使用以下代码执行与区域设置无关的测试，以确定 char * ext 是否匹配 "gif" 忽略大写/小写差异：

```
#include <windows.h>
int f(char *ext)
{
  // code...
  return (CompareString(
                        LOCALE_INVARIANT,
                        NORM_IGNORECASE,
                        ext,
                        -1,
                        TEXT ("gif"),
                        -1) == CSTR_EQUAL);
}
```
