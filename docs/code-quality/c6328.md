# <a name="c6328"></a>C6328

警告 C6328：大小不匹配：作为参数传递 @no__t > \<number > 在调用 \<function 时需要 > 2type @no__t

将类型为 `char` 的参数传递给 C 运行时，名为 @no__t 的基于字符的例程（例如 `isspace()`）可能会产生不可预知的结果。 例如，如果类型为 `char` 的 SBCS 或 MBCS 单字节字符的值大于 `0x7F`，则该值为负值。 如果传递的是 @no__t 0，则编译器可能会将值转换为有符号 @no__t 1 或已签名 `long`。 此值可以由编译器进行符号扩展，出现意外的结果。 例如，`isspace` 接受类型 @no__t 的参数-1;但是，其输入参数的有效值范围为：

`0 <= c <= 255`，加上特殊值 `EOF`。

## <a name="example"></a>示例

默认情况下，`char` 在 Visual C++ 中为有符号类型，所以 char 类型的变量的值范围是 `-128 <= c <= 127`。 因此，如果你执行了以下操作，`c` 会将签名扩展到签名的 `int`，其值为-37，这超出了[isspace](/cpp/standard-library/locale-functions#isspace)的有效范围：

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( c );
    // code ...
}
```

若要更正此问题，请使用 `static_cast`，如下面的代码所示：

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( static_cast<unsigned char> (c) );
    // code ...
}
```

警告 C6328 专门用于捕获此 bug。 对于7位 ASCII 范围内的字符，不需要强制转换。 超出该范围的字符可能会产生不可预知的结果，例如，程序错误和终止。
