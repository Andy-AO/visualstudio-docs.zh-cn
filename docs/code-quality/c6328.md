# <a name="c6328"></a>C6328

警告 C6328：大小不匹配\<：如果在调用\<函数\<时\<需要类型 >，则类型 > 作为参数号传递 > >

将类型`char`的参数传递给 C 运行时，名为`is<xxx>()`的基于字符的例程（ `isspace()`如）可能会产生不可预知的结果。 例如，如果类型为 `char` 的 SBCS 或 MBCS 单字节字符的值大于 `0x7F`，则该值为负值。 如果传递`char`了，编译器可能会将值转换为已签名`int`或已签名`long`。 此值可以由编译器进行符号扩展，出现意外的结果。 例如， `isspace`接受类型`int`的参数; 但是，其输入参数的值的有效范围是：

`0 <= c <= 255`，加上特殊值 `EOF`。

## <a name="example"></a>示例

默认情况下，`char` 在 Visual C++ 中为有符号类型，所以 char 类型的变量的值范围是 `-128 <= c <= 127`。 因此，如果你执行了以下操作`c` ，则会将签名扩展到值`int`为-37 （在[isspace](/cpp/standard-library/locale-functions#isspace)的有效范围之外）的签名。

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( c );
    // code ...
}
```

若要更正此问题，请使用 `static_cast`，如下面的代码所示：

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( static_cast<unsigned char> (c) );
    // code ...
}
```

警告 C6328 专门用于捕获此 bug。 对于7位 ASCII 范围内的字符，不需要强制转换。 超出该范围的字符可能会产生不可预知的结果，例如，程序错误和终止。
