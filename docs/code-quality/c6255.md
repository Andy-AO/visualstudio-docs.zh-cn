# <a name="c6255"></a>C6255
警告 C6255： _alloca 通过引发堆栈溢出异常来指示失败。 请考虑改用 _malloca

 此警告意味着在本地异常处理之外检测到对 @no__t 的调用。 应始终从异常处理程序的受保护范围内调用 `_alloca`，因为它可能在失败时引发堆栈溢出异常。 如果可能，请考虑使用 `_malloca`，而不是使用 @no__t，这是 @no__t 的更安全的版本。

## <a name="example"></a>示例
 下面的代码将生成此警告，因为 `_alloca` 会生成异常：

```
#include <windows.h>

void f( )
{
  void *p = _alloca(10);
  // code ...
}
```

 若要更正此警告，请使用 `_malloca` 并添加异常处理程序，如以下代码所示：

```
#include <windows.h>
#include <malloc.h>

void f( )
{
  void *p;
  int errcode;
  __try
  {
    p = _malloca(10);
    // code...
    _freea(p);
  }
  __except( (GetExceptionCode() == STATUS_STACK_OVERFLOW ) ?
              EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH )
  {
    errcode = _resetstkoflw();
    // code ...
  }
}
```
