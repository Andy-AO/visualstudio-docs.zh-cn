# <a name="c26404--dont_delete_invalid"></a>C26404  DONT_DELETE_INVALID
一旦所有者指针释放或传输其资源，就会进入 "无效" 状态。
删除此类指针可能导致立即导致内存损坏，原因是双重删除，或从另一个所有者指针访问已删除的资源时出现访问冲突。

## <a name="example-1-deleting-an-owner-after-transferring-its-value"></a>示例 1：在转移值后删除所有者

```cpp
gsl::owner<State*> validState = nullptr;
gsl::owner<State*> state = ReadState();
validState = state;
if (!IsValid(state))
    delete state;   // C26404
```

## <a name="example-2-deleting-an-uninitialized-owner"></a>示例 2：删除未初始化的所有者

```cpp
gsl::owner<Message*> message;
if (popLast)
    message = ReleaseMessage();
delete message; // C26404
```
