# <a name="c6220"></a>C6220
警告 C6220-语义不同的整数类型之间的隐式强制转换：将 HRESULT 与-1 进行比较。 请考虑改用 SUCCEEDED 或 FAILED 宏

 此警告意味着将 `HRESULT` 与不是格式正确的-1 的显式非 @no__t 值-@no__t 1 进行比较。 @No__t-0 （`E_FAIL`）中的失败不是由-1 表示。 因此，对整数 @no__t 0 的隐式强制转换将生成不正确的值，并且可能导致错误的结果。

## <a name="example"></a>示例
 在大多数情况下，此警告是由于代码错误地要求返回 `HRESULT` 的函数改为通过使用-1 作为失败值返回整数。 下面的代码示例将生成此警告：

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == -1)
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 最好使用 @no__t 0 或 `FAILED` 宏来测试 @no__t 的值。 若要更正此警告，请使用以下代码：

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 对于此警告，@no__t 0 类型等效于 `HRESULT`。

 显式比较适用于检查特定 `HRESULT` 值，如 `E_FAIL`。 否则，请使用 `SUCCEEDED` 或 `FAILED` 宏。

 有关详细信息，请参阅[SUCCEEDED 宏](http://go.microsoft.com/fwlink/?LinkId=92738)和[FAILED 宏](http://go.microsoft.com/fwlink/?LinkId=180875)。

 请注意，使用 malloc 和 free （以及相关的动态内存分配 Api）对内存泄漏和异常有许多缺陷。 若要避免这种类型的泄漏和异常问题，请使用C++标准模板库（STL）提供的机制。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)、 [unique_ptr](/cpp/standard-library/unique-ptr-class)和[矢量](/cpp/standard-library/vector)。 有关详细信息，请参阅[智能指针](/cpp/cpp/smart-pointers-modern-cpp)和[ C++标准库](/cpp/standard-library/cpp-standard-library-reference)。
