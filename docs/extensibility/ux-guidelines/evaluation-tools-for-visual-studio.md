# <a name="evaluation-tools-for-visual-studio"></a>Visual Studio 的计算工具
## <a name="craftsmanship-checklist-for-visual-studio"></a>视觉工作室的工艺清单
 使用此检查表可评估用户体验质量，了解视觉和交互详细信息。

### <a name="overview"></a>概述

- 验证所有命令是否导致反馈，告知用户其命令已执行。

- 验证所有 UI 元素和控件在所有主题和高对比度模式下是否可见。

- 验证非活动选择和活动选择在标准和高对比度模式下始终有区别。

- 验证焦点是否始终可见且明显。

### <a name="performance"></a>性能

- 验证如果命令需要超过一秒才能完成，则显示某种"忙"指示器。

- 验证如果命令需要超过 10 秒才能完成，将显示一个显式进度栏，确定（首选）或不确定。

### <a name="ui-text"></a>UI 文本

- 验证所有标签是否都是句子或标题大小写，并且没有文本是完全小写。

    ||正确|不正确|
    |-|-------------|---------------|
    |**命令文本（全部）**|判刑案例：<br /><br /> **目录名称：**|目录名称：|
    |**按钮文本（客户端）**|标题案例：<br /><br /> **[ 设置为默认值 ]**|设置为默认值|
    |**按钮文本（在线）**|判刑案例：<br /><br /> **[ 设置为默认值 ]**||

- 验证除组标题和按钮之外的所有标签都以冒号结尾，并在与其配对的控件之前结束。

- 验证启动 UI 以在省略号中捕获用户输入端的按钮、命令和命令链接 **[...]**.

  示例：

  - 对话框上的 **[高级...]** 按钮。

  - "工具"菜单（**工具>选项**）下的命令选项不应得到省略号，因为启动对话框本身是命令的意图。

- 验证 UI 是否不包含缩写，但行业标准术语除外。 例如，HTML 和 TCP/IP 都不需要拼写出来，尽管 OOM（内存不足）和 PII（个人身份信息）应该。

### <a name="keyboard-access"></a>键盘访问

- 验证是否有使用键盘完成每个任务的方法。 通常，这是通过每个控件的键盘访问来完成的，但对于某些高度可视的区域，诸如访问代码视图等解决方法是可以接受的。

- 验证是否可以按逻辑顺序（从左到右和从上到下）对控件进行选项卡式检查。 虽然这对于大多数控件都是最佳做法，但并非所有控件都需要此方法。 例如，验证单选按钮控件是否位于具有单个制表位的组中。

- 验证所有控件是否具有标签，并且每个标签都有助记符（例外情况包括一些可能遵循选项卡中标记控件的非标记控件）。

- 验证不存在助记符冲突。

### <a name="fonts"></a>字体

- 验证所有字体（面部、大小、颜色）是否一致使用并维护层次结构。

- 验证所有 UI 元素是否使用环境字体服务。 （请参阅[可视化工作室的字体和格式](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)）

     要检查服务是否正在使用，请转到**工具>选项>字体和颜色**。 在"设置"下拉下，选择"环境字体"并将字体更改为风格上有所不同的内容（如哈灵顿或漫画圣人），并将大小设置为 12 磅。 然后，单击“确定”。 您可能需要重新启动 IDE，但大多数 UI 将立即更改。 即使在重新启动时也不会拾取字体更改的区域不使用环境字体。

- 验证作为服务派生的字体（例如，粗体或放大文本）在环境字体大小更改时保留其大小和格式与"正常"文本相关。

- 验证由于字体放大，没有剪切错误。 剪切的字体可能是固定高度控件或固定高度容器的结果。

### <a name="dialogs"></a>对话框

- 验证对话框标题是否与启动对话框的标题的命令相同。

- 验证所有标准控件是否与操作系统一致：背景颜色是标准控件，任何控件都不应具有特殊的重新模板化样式，使其看起来不同于标准控件。

- 验证窗体中的边距应为 12 像素，并且应显示均匀且一致。

- 验证对话框是否显示在 IDE 外壳或生成对话框的窗口中居中。

- 有用时，对话框应可调整大小。 对于可调整大小的对话框，请验证调整大小时，适当的控件必须调整大小，而对话框的其他部分保持不变。

- 验证可调整大小的对话框是否保留任何用户调整的大小（大小、位置、对话框控件的扩展等）。

- 验证标题栏中没有图标。

- 验证标题栏中没有"最小化"和"最大化"按钮。

#### <a name="dialog-operation-buttons-vs-client-only"></a>对话框操作按钮（仅限 VS 客户端）

- 验证操作按钮是否按此顺序排列：**确定**、**取消**、**应用**。

- 验证 **"确定****"和"取消"** 按钮是否为标准大小：75x23 像素。

- 验证 **"确定****"和"取消"** 按钮的大小是否相等，而不考虑字符串长度。

- 如果操作按钮上的标签要求按钮比标准宽，请验证相应的 **"取消"** 按钮的大小是否相同。

- 验证按钮和相关控件之间是否有 6 像素填充。

- 验证 **"确定**"和 **"取消"** 按钮没有助记符（由带下划线字母定义的访问键）。

- 验证一个按钮（通常**为"确定**"）默认具有焦点。

- 验证**Esc**是否取消对话框

- 如果焦点不在处理 Enter 的控件中，请验证**Enter**是否执行默认按钮。

- 验证 **"确定**"和 **"取消"** 按钮是否位于对话框的右下角。 在极少数例外情况中，它们垂直堆叠在右上角是可以接受的。

- 仅当对话框中的其他按钮处于水平对齐时，才验证垂直配置是否使用。

### <a name="control-standards"></a>控制标准

#### <a name="general"></a>常规

- 验证是否尽可能有良好的默认值，以加快用户交互，并引导用户获得安全或共同的结果。

- 验证标准控件是否以相同的方式操作，以便用户根据以前的体验知道将会发生什么。

#### <a name="label-controls"></a>标签控件

- 验证每个控件是否具有标签，并且每个标签在视觉上与其控件配对（通常在 4-6 像素范围内），并且比其他控件更接近其相应的控件。

- 如果定位在上方并水平居中，则验证标注是否与控件左边缘对齐左，以便标签的基线与输入文本的基线对齐（如果位于左侧）。

- 验证如果多个堆叠标签和输入控件位于控件的左侧，则标签向左刷新，与对话框边缘的距离相等，切勿向右刷新，并且与控件的距离相等。 对应均匀分布，除非它们需要额外的间距来指示分组。

#### <a name="input-controls-text-boxes-and-combo-boxes"></a>输入控件（文本框和组合框）

- 验证使用默认环境字体时，文本框、组合框和按钮的显示高度均为 23 像素。

- 如果使用提示文本，请验证颜色是否设置为`Environment.ControlEditHintText`使用颜色服务。

- 如果该字段是必须标识为该字段的必填字段，请验证：

  - 背景设置为`Environment.ControlEditRequiredBackground`，前景设置为`Environment.ControlEditRequiredHintText`

  - 控件中有显示为 **"必需>"\<** 的提示文本

#### <a name="button-controls"></a>按钮控件

- 验证按钮的最小大小是否为 75x23 像素，除非容纳较长的文本。

- 验证按钮的左右边距是否为 3-5 像素，以及内容填充。

- 可以使用一个小方形按钮，其中只有省略号 **[...] 而不是** **[Browse...]** 按钮（或类似功能）。 如果使用，请验证按钮的大小是否为 23x23。

- 如果对话框中有多个 **[Browse...]** 按钮，则验证缩短的版本（仅限省略号 **[...]**） 是否用于所有内容。

- 验证省略**号 [...]** 按钮没有助记符。 当焦点位于其旁边的输入控件上时，一个选项卡应将焦点移动到椭圆按钮。

- 验证启动捕获更多用户输入的辅助 UI 的按钮、命令和命令链接必须以省略号 **[...]** 结尾。

#### <a name="hyperlinks"></a>超链接

- 验证超链接控件在激活时是否从不闪烁红色。 这是一个指示，表明颜色服务未使用

- 验证所使用的 VS 颜色是否：

  - `Environment.ControlLinkText`

  - `Environment.ControlLinkTextHover`

  - `Environment.ControlLinkTextPressed`

- 验证超链接是否显示为蓝色，除非嵌入段落，否则没有下划线。

#### <a name="check-boxes"></a>复选框

- 如果复选框具有多行文本，请验证该框是否与第一行文本对齐，而不是在所有行中垂直居中。

- 验证复选框始终指示二进制选择，并且不导航用户或打开新窗口或页面。

- 如果复选框显示与输入控件相关的选项，请验证该选项位于该控件的左侧和非常接近的位置，以指示其关系。

- 验证复选框**是否从未**用作启用对话框或页面的全部内容的方法。

#### <a name="group-boxes"></a>组框

- 验证对话框中不包含包含对话框全部内容的单个组框。

- 验证每个组框中至少有两个控件。

- 对话框中很少应该有两个以上组框。

- 验证没有嵌套组框。

### <a name="icons"></a>图标

- 验证图标在深色主题中是否正确反转。

- 验证所有图标是否基于核心概念。

- 验证每个图标是否不同、易于识别且不包含两个以上概念（没有状态修改器/语言）。

- 验证基本图标是否显示在空间内的中心。

- 验证所有图标在高对比度模式下是否清晰。

- 验证使用的任何颜色是否符合颜色使用标准。

- 验证图标周围没有光晕（边框）。 （如果存在，光晕应与相邻 UI 的背景颜色匹配）。

### <a name="touch-enabled-ui"></a>启用触摸的 UI

- 验证交互式控件是否足够大，易于触摸 - 最小**23x23 像素**大小

- 验证最常用的控件的大小是否至少**为 40x40 像素**。

- 验证交互式控件之间至少有**5 像素的间距**
