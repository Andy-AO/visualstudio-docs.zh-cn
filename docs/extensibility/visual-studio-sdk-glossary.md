# <a name="visual-studio-sdk-glossary"></a>可视化工作室 SDK 词汇表
此术语表为[!INCLUDE[vsipsdk](../extensibility/includes/vsipsdk_md.md)]文档中使用的术语提供定义。

## <a name="terms"></a>术语
 外接程序 添加到主应用程序的实用程序应用程序、驱动程序或其他软件。 在 Visual Studio 集成开发环境 （IDE） 中，外接程序是基于自动化的应用程序，可扩展 IDE 的功能。

 自动化模型在 Visual Studio 的早期版本中称为扩展性模型，是一个编程接口，允许您访问驱动 IDE 的基础例程。 加载项、向导和宏使用自动化模型中的对象来控制或扩展 IDE 的功能。

 命令 UI 上下文与 UI 命令或元素（如工具栏）的可见性关联。 命令 UI 上下文与选择上下文不同，因为它不附加到窗口。

 命令 UI 上下文可用于：

- 将 GUID 分配给激活特定窗口时显示的工具栏。

- 将 GUID 分配给命令的可用性，而无需加载或运行 VSPackage。

- 分配 GUID 以影响活动密钥绑定。

- 分配 GUID 以打开宏录制。

- 分配 GUID 以激活调试模式或在编辑器中的设计和运行模式之间切换。

  组件软件，可以成为应用程序功能的一部分，而该应用程序没有任何关于软件实现的任何预先存在的信息。 组件和应用程序之间的通信仅通过 OLE 样式接口进行。

  组件管理器 A`SOleComponentManager`服务 ，它为顶级组件提供非用户界面协调服务。 服务`SOleComponentManager`实现接口`IOleComponentManager`。

  组件 UI 管理器`SOleComponentUIManager`A 服务，提供用户界面协调服务。 服务`SOleComponentUIManager`实现 和`IOleComponentUIManager``IOleInPlaceComponentUIManager`接口。

  上下文包`IVsUserContext`附加到环境组件的对象 （COM 对象）。 此对象包含查找关键字 **、F1**关键字和与组件相关的属性。 此外，上下文包还会指向链接到它们的任何子上下文包。

  上下文提供程序 IDE 中具有与其关联的上下文包的组件。 此类组件包括工具窗口、编辑器或项目层次结构。

  设计器允许用户操作 UI 的元素（窗体、按钮和其他控件）的编程接口。

  DocData A COM 对象封装文档的基础数据，其中存在文档/视图分离（例如，在文本编辑器案例中，这将是所有文本编辑器视图的基础文本缓冲区）。 如果编辑器工厂不提供此对象，IDE 将代表它制造一个对象。 此对象的责任是管理数据持久性和共享语义的多个视图在此相同`DocData`。 如果`DocData`对象支持`IOleCommandTarget`接口，则该接口包含在 UIShell 的命令路由中。

  文档对象技术用于在主机提供的帧中托管 UI。 更具体地说，此术语是指支持`IOleDocument`和相关接口的任何嵌入。 该技术具有许多潜在的应用，如 COM 文档的实现详细信息、Visual Basic 5.0 中的工具窗口、Visual Basic 6.0 中的 ActiveX 设计器等。

  文档 用于将文档作为一个整体进行泛泛引用 ，`DocData`和`DocView`。 例如，DocumentFrame 包含 ，`DocView`但它也保留对`DocData`处理持久性的 引用。

  文档查看用户与之交互的文档对象/嵌入/窗口窗格以查看和操作基础`DocData`。 用户不会利用`DocObject`作为界面设计一部分的文档/视图分离。 用户使用整个 DocObject 充当视图，而不是使用称为 的基础数据更抽象（不太正式）的概念`DocData`。 `DocView`对象始终嵌入 IDE 的文档框对象（MDI 子窗口）。

  `DTE` DTE（开发工具扩展性）对象是 Visual Studio 自动化模型中访问点最多的对象，它允许您以编程方式自动执行和扩展 IDE。

  由 IDE 实现的动态帮助窗口工具窗口，并显示查找关键字或**F1**帮助主题的列表。

  实现 的`DocView`编辑器代码（类、CLSID） 如果支持视图`DocData`和数据分离，它也会实现。

  扩展修改、自定义或添加到 IDE 的功能。 使用自动化模型或 VSPackages 创建扩展。

  外部编辑器 不是特定于 IDE 的编辑器，如 Microsoft Word。 它已通过自己的机制注册，可在 IDE 之外使用。 如果可以嵌入此编辑器，它将显示在 IDE 窗口中。 如果无法嵌入，则创建单独的顶级窗口。

  层次结构节点树，每个节点与一组属性关联。

  独立的顶级组件一个组件，它使用无模式的顶级窗口，可以有效地作为独立应用程序窗口运行，但作为进程内对象实现。 因此，独立的顶级组件必须协调模式和消息循环服务与 IDE。 进程内对象没有自己的消息循环。

  信息提供程序 信息提供程序是一个模块，可以查找关键字并返回主题列表，以`IVsUserContextItem`对象的形式。 要为信息提供商提供**F1**和查找关键字项，请注册已编译的帮助文件 *（。HxS*） 与系统。 这些文件中的"帮助"主题提供动态帮助窗口中显示的主题列表，并显示用户是否按**F1**。

  就地组件 VSPackage 对象，该对象`IOleInPlaceComponent`实现接口来管理视觉上包含在 IDE 拥有的文档窗口中的窗口。 就地组件不参与标准 OLE 菜单合并;相反，他们将其用户界面元素集成到 IDE 中。

  有两种类型的就地组件：硬接线就地组件和组件控件。

  硬连线就地组件具有菜单、工具栏和命令，它们紧密集成到 IDE 的用户界面中，看起来好像它们直接内置到 IDE 中。

  组件控件没有将任何自己的用户界面元素集成到 IDE 中;因此，组件控件没有将任何用户界面元素集成到 IDE 中。相反，他们使用 IDE 的菜单、命令和工具栏。 例如，"粗体"命令可用于在嵌入窗体的富文本控件中粗制所选单词。 但是，组件控件可以请求显示动态安装的特定于组件的 UI 元素。

  语言服务 一组对象，允许 VSPackage 开发人员实现计算机语言代码编辑器的功能，如文本标记和着色。

  杂项文件项目项目项目用于容纳不在任何项目中的打开文件。 此项目中的项目列表不会持久化。

  项目项目由层次结构对象或实现接口的`IVsHierarchy`COM 对象组成。

  项目特定的设计器或编辑器不能独立于项目类型的设计器。 所有特定于项目的设计人员都必须在注册表中输入其编辑器工厂信息。 然后，每当在特定项目中打开某个文件类型时，IDE 就可以实例化设计器。

  项目类型窗口 不断跟踪当前活动项目层次结构和全局选择上下文中的项的窗口。 项目类型窗口使用`SVsTrackSelectionEx`服务来提醒 IDE 更改并显示向用户的反馈。 解决方案资源管理器是项目类型窗口的示例。

  属性窗口以前属性浏览器。

  基于引用的项目项目，它不需要项目的文件位于同一目录中。 相反，对来自其他不相关目录的文件的引用由项目本身存储和维护。

  运行文档表 内部结构，IDE 通过该结构维护所有当前打开的文档的列表。 该列表包括内存中的所有打开的文档，而不管当前是否正在编辑这些文档。 文档是保存的任何项，包括在编辑器中打开的存储过程、项目中的文件或主项目文件（例如，*.vcproj 文件）。

  选择上下文 数据，它是 IDE 中每个窗口的详细信息的一部分，用于跟踪活动选择。 选择上下文包括：

- 指向项目层次结构`IVsHierarchy`接口的指针

- 项目项的项标识符。

- 指向提供对`ISelectionContainer`活动对象属性访问的接口的指针。

- 元素值的数组。

  服务 一组 COM 接口的协定，这些接口位于单个 COM 对象中。 创建由 GUID 标识的服务时，可以定义执行服务的 COM 接口集。 COM 对象使用服务相互通信。

  解决方案 用户使用的相关项目组。

  标准设计器一种设计器，可以独立于项目类型使用。 所有标准设计人员必须在注册表中输入其编辑器工厂信息。 然后，每当打开具有特定扩展名的文件时，IDE 就可以实例化设计器。 数据必须保存到文件中。

  标准编辑器，可独立于任何特定项目类型使用。 此类编辑器在注册表中注册了编辑器工厂。 这允许 IDE 查找和调用编辑器。

  标准操作系统编辑器不是特定于可视化工作室的嵌入。 它是使用众所周知的 Win32 密钥注册的（例如，Win32 资源管理器知道如何调用）。 如果可以嵌入这样的编辑器，编辑器仍然显示在 IDE 中的位置。 否则，将为此编辑器创建单独的顶级窗口。

  子上下文包`IVsUserContext`链接到上下文包的对象。 对象包含 IDE 组件中所选内容的查找关键字 **、F1**关键字和属性。 子上下文的示例包括工具窗口中的命令或编辑器中的关键字。

  任务列表 由 IDE 实现并显示活动任务的列表的工具窗口。

  文本缓冲区 对象`VSTextBuffer`的常见名称 。

  文本视图 对象`VSTextView`的常见名称 。

  工具顶级组件一个组件，作为无模式弹出窗口运行，与 IDE 的用户界面紧密协调。 与独立的顶级组件一样，工具顶级组件还必须将模式和消息循环服务与 IDE 协调。

  顶级组件 A VSPackage 对象，用于管理无模式的顶级窗口，而不是 IDE 窗口的工作区。 顶级组件实现`IOleComponent`接口，以利用消息循环服务，如访问空闲时间。

  UI 活动 A VSPackage 对象可见且当前具有焦点。

  UI 层次结构实现`IVsUIHierarchy`接口以允许显示层次结构的 COM 对象。 UI 层次结构窗口实现`ISelectionContainer`接口以更新属性窗口;如果需要，其他项目类型窗口可以使用此实现。

  VSCT 可视化工作室命令表。 .vsct 文件包含有关菜单、工具栏和命令以 XML 格式放置和行为的信息。

  VSPackage 可安装的软件，通过贡献以下一个或多个项目来扩展 Visual Studio IDE：用户界面、服务、项目类型或编辑器/设计器。 VSPackage 由实现`IVsPackage`接口的 COM 对象以及实现其他接口以支持选择和其他功能的一个或多个其他 COM 对象组成。 此外，VSPackage 具有特定的注册要求。
