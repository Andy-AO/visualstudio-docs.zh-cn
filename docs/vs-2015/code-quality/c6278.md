# <a name="c6278"></a>C6278
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 >c6278:\<变量 > 是使用数组 new [] 分配，但使用标量 delete 删除。 将不会调用析构函数  
  
 此警告只出现在C++代码，并指示调用的函数具有不一致分配内存以及数组**new []** 运算符，但释放它使用标量**删除**运算符。 这是未定义的行为，根据C++标准和 Microsoft VisualC++实现。 这是可能导致问题的至少三个原因有：  
  
- 数组中的单个对象的构造函数进行调用，但不是会调用析构函数。  
  
- 如果全局，或特定于类**运算符 new**并**运算符 delete**不符合**运算符 new []** 和**运算符 delete []**，则可能会出现意外的结果。  
  
- 它始终是十分危险，依赖于未定义的行为。  
  
  此缺陷的确切的后果难以预测。 它可能会导致泄漏; 如果类具有执行取消分配内存; 的析构函数对于具有析构函数的执行一些在语义上是重要的操作; 类不一致的行为或内存损坏和崩溃时运算符被重写。 在其他情况下可能并不重要，具体取决于编译器及其库的实现不匹配。 分析工具不能始终区分这些情况。  
  
  如果使用数组分配内存**new []**，它应通常用来释放数组**delete []**。  
  
## <a name="example"></a>示例  
 下面的示例代码将生成此警告：  
  
```  
class A  
{  
  // members  
};  
  
void f( )  
{  
  A *pA = new A[5];  
  // code ...  
  delete pA;  
}  
```  
  
 若要更正此警告，请使用下面的示例代码：  
  
```  
void f( )  
{  
  A *pA = new A[5];  
  // code ...  
  delete[] pA;  
}  
```  
  
 如果数组中的基础对象是基元类型如`int`， `float`， `enum`，或指针，有不析构函数的调用。 在这些情况下，警告[C6283](../code-quality/c6283.md)改为报告。  
  
 使用新和删除已在内存泄漏和异常方面的许多缺陷。 若要避免这些类型的泄漏和异常问题完全，使用的机制，通过提供C++标准模板库 (STL)。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，并[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 有关详细信息，请参阅[智能指针](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)并[C++标准库](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
