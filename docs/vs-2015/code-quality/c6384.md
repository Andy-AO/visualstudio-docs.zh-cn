# <a name="c6384"></a>C6384
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6384： 另一个值除以 sizeof 指针  
  
 此警告表明大小计算可能不正确。 若要计算数组中的元素数，1 有时整除的第一个元素; 大小的数组的大小但是，如果阵列为实际的指针，结果会比预期通常有所不同。  
  
 如果指针是一个函数参数，而且不传递缓冲区的大小，则不可以计算可用的最大缓冲区。 本地分配指针后，应使用分配中使用的大小。  
  
## <a name="example"></a>示例  
 下面的代码生成此警告：  
  
```  
#include <windows.h>  
#include <TCHAR.h>  
  
#define SIZE 15  
  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
    _tcsncpy(dest, src, sizeof dest  / sizeof dest[0]);   
  }  
}  
```  
  
 若要更正此警告，请将传递的缓冲区大小，如下面的代码中所示：  
  
```  
#include <windows.h>  
#include <TCHAR.h>  
  
#define SIZE 15  
  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
    _tcsncpy(dest, src, SIZE);  
  }  
}  
```  
  
 若要更正使用安全的字符串函数 _tcsncpy_s 此警告，请使用以下代码：  
  
```  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
   _tcsncpy_s(dest, SIZE, src, SIZE);  
  }  
}   
```  
  
 请注意，使用新和删除已在内存泄漏和异常方面的许多缺陷。 若要避免这些类型的泄漏和异常问题完全，使用的机制，通过提供C++标准模板库 (STL)。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，并[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 有关详细信息，请参阅[智能指针](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)并[C++标准库](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
