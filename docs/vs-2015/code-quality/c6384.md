# <a name="c6384"></a>C6384
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6384：将指针除以另一个值  
  
 此警告表明大小计算可能不正确。 若要计算数组中的元素数，则有时会将数组的大小除以第一个元素的大小;但是，当数组实际上是一个指针时，结果通常与预期的不同。  
  
 如果指针是函数参数并且未传递缓冲区的大小，则无法计算可用的最大缓冲区数。 在本地分配指针时，应使用分配中使用的大小。  
  
## <a name="example"></a>示例  
 下面的代码将生成此警告：  
  
```  
#include <windows.h>  
#include <TCHAR.h>  
  
#define SIZE 15  
  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
    _tcsncpy(dest, src, sizeof dest  / sizeof dest[0]);   
  }  
}  
```  
  
 若要更正此警告，请传递缓冲区大小，如以下代码所示：  
  
```  
#include <windows.h>  
#include <TCHAR.h>  
  
#define SIZE 15  
  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
    _tcsncpy(dest, src, SIZE);  
  }  
}  
```  
  
 若要使用安全字符串函数 _tcsncpy_s 更正此警告，请使用以下代码：  
  
```  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
   _tcsncpy_s(dest, SIZE, src, SIZE);  
  }  
}   
```  
  
 请注意，在内存泄漏和异常方面，使用 new 和 delete 具有许多缺陷。 若要避免这种类型的泄漏和异常问题，请使用 c + + 标准模板库 (STL) 提供的机制。 其中包括 [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)、 [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)和 [矢量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 有关详细信息，请参阅 [智能指针](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) 和 [c + + 标准库](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
