# <a name="c6279"></a>C6279
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6279:\<变量 > 使用标量 new，使用数组 delete [] 删除分配  
  
 此警告只出现在C++代码，并指示，调用的函数不一致所分配的内存使用标量**新**运算符，但用来释放它数组**delete []** 运算符。 如果内存分配使用标量**新**，通常应使用标量释放**删除**。  
  
 这是可能导致问题的至少三个原因有：  
  
- 数组中的单个对象的构造函数不会调用的尽管析构函数。  
  
- 如果全局 （或类特定于）**运算符 new**并**运算符 delete**不符合**运算符 new []** 和**运算符 delete []**，则可能会出现意外的结果。  
  
  此缺陷的确切的后果难以预测。 因为不会调用构造函数，它可能会导致随机行为或由于未初始化的内存使用率而导致系统崩溃。 或者，它可能会导致内存分配和故障情况下在被重写运算符。 在极少数情况下，不匹配可能并不重要。 分析工具不当前区分这些情况。  
  
## <a name="example"></a>示例  
 下面的代码生成此警告：  
  
```cpp  
class A  
{  
  // members  
};  
  
void f ( )  
{  
  A *pA = new A;  
  //code ...  
  delete[] pA;  
}  
```  
  
 若要更正此警告，请使用以下代码：  
  
```cpp  
void f( )  
{  
  A *pA = new A;  
  //code ...  
  delete pA;  
}  
```  
  
 若要完全避免这些分配问题，请使用 C++ 标准模板库 (STL) 提供的机制。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，并[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 有关详细信息，请参阅[智能指针](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)并[C++标准库](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
