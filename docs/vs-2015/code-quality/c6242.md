# <a name="c6242"></a>C6242
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6242:从此 try 块跳转将强制展开局部堆栈。 会产生严重影响性能  
  
 此警告表明，跳转语句会导致退出受保护的块的的 try-finally 以外的其他通过贯穿的控制流。  
  
 保留受保护的的 try-finally 块其他不是通过从上次故障语句需要本地展开发生。 展开局部堆栈通常需要大约 1000年机器指令;因此，它是对性能造成不利影响。  
  
 使用`_leave`退出受保护的的 try-finally 块。  
  
## <a name="example"></a>示例  
 下面的代码生成此警告：  
  
```  
#include <malloc.h>  
void DoSomething(char *p); // function can throw exception  
  
int f( )  
{  
   char *ptr = 0;  
   __try   
   {  
      ptr = (char*) malloc(10);  
      if ( !ptr )   
      {  
         return 0;   //Warning: 6242  
      }  
      DoSomething( ptr );  
   }  
   __finally   
   {  
      free( ptr );  
   }  
   return 1;  
}  
```  
  
 若要更正此警告，请使用`__leave`如下面的代码中所示：  
  
```  
#include <malloc.h>  
void DoSomething(char *p);   
int f()  
{  
   char *ptr = 0;  
   int retVal = 0;  
  
   __try   
   {  
      ptr = (char *) malloc(10);  
      if ( !ptr )  
      {  
         retVal = 0;  
         __leave;   //No warning  
      }  
      DoSomething( ptr );  
      retVal = 1;  
   }  
   __finally  
   {  
      free( ptr );  
   }  
  
   return retVal;  
}  
```  
  
 使用 malloc 和免费有内存泄漏和异常方面的许多缺陷。 若要避免这些类型的泄漏和异常问题完全，使用的机制，通过提供C++标准模板库 (STL)。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，并[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 有关详细信息，请参阅[智能指针](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)并[C++标准库](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
