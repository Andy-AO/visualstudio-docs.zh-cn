# <a name="how-to-create-a-work-item-for-a-managed-code-defect"></a>如何：为托管代码缺陷创建工作项
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

您可以使用工作项跟踪功能从中记录工作项 [!INCLUDE[vsPreLong](../includes/vsprelong-md.md)] 。 若要使用此功能，你的项目必须是中的团队项目的一部分 [!INCLUDE[esprfound](../includes/esprfound-md.md)] 。

### <a name="to-create-a-work-item-for-managed-code-defect"></a>为托管代码缺陷创建工作项

1. 在 " **代码分析** " 窗口中，选择警告。

2. 选择 " **操作**"，然后选择 " **创建工作项** "，并选择要创建的工作项的类型。

     系统将创建一个新的工作项来指定缺陷信息。

### <a name="to-create-a-work-item-for-multiple-managed-code-defects"></a>为多个托管代码缺陷创建工作项

1. 在 **错误列表**中，选择 "多个警告"，然后右键单击警告。

2. 指向 " **创建工作项** "，然后单击要创建的工作项的类型。

     将为所选的所有警告创建一个工作项，以指定 bug 信息。
