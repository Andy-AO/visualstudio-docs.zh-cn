# <a name="c6226"></a>C6226
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
警告 C6226:语义不同的整数类型之间的隐式强制转换： 将-1 赋给 HRESULT。 请考虑改用 E_FAIL。

此警告意味着`HRESULT`分配或初始化为显式值为-1。 此警告通常由意外的整数的混淆和`HRESULT`类型。 若要指示成功，请使用符号常量`S_OK`相反。 若要指示失败，请使用开始 E_ 的符号常量*常量*，如`E_FAIL`。

有关详细信息，请参阅[SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded)并[失败](/windows/desktop/api/winerror/nf-winerror-failed)宏。

## <a name="example"></a>示例

下面的代码生成此警告：

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = -1;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

若要更正此警告，请使用以下代码：

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = E_FAIL;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

此警告，请`SCODE`类型会被视为`HRESULT`。

利用`malloc`和`free`（和相关的动态内存的 Api） 为内存泄漏和异常的原因有许多缺陷。 若要避免这些类型的泄漏和异常问题，请使用提供的指针和容器类C++标准库。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)， [unique_ptr](/cpp/standard-library/unique-ptr-class)，并[向量](/cpp/standard-library/vector)。 有关详细信息，请参阅[智能指针](/cpp/cpp/smart-pointers-modern-cpp)并[C++标准库](/cpp/standard-library/cpp-standard-library-reference)。
